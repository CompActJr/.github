## 🤝 Colaboração na Comunidade

Para manter um ambiente saudável e produtivo, todos os colaboradores devem seguir estas regras:
    **Respeito Mútuo:** Tratamos todos com profissionalismo e cortesia.
    **Qualidade:** Priorize código limpo (Clean Code) e bem testado.
    **Comunicação:** Utilize os canais oficiais e as Issues dos projetos para reportar problemas ou sugerir melhorias além de se atualizar com suas tasks se for um desenvolvedor
    **Padronização** siga convenções em commits e branchs com nome de sua feature e ou /nome do repo na branch,
     
## 🔑 Configuração de SSH no GitHub
    
Para colaborar, recomendamos o uso de chaves SSH para maior segurança e praticidade.
      
### No Linux 🐧

Abra o terminal e gere a chave: `ssh-keygen -t ed25519 -C "seu_email@exemplo.com"`
Inicie o ssh-agent (por padrão o linux já tem ele rodando mas verifique): `eval "$(ssh-agent -s)"`
Adicione a chave: `ssh-add ~/.ssh/id_ed25519`
Copie a chave pública: `cat ~/.ssh/id_ed25519.pub`
Cole no GitHub em: **Settings > SSH and GPG keys > New SSH key**.
 
### No Windows 🪟
1. Abra o **Git Bash**.
2. Siga os mesmos passos do Linux acima.
3. Se preferir usar o PowerShell, certifique-se de que o serviço `ssh-agent` está ativo.
  
## 📐 Padrões de Desenvolvimento
     
### 🌿 Branches

Utilizamos o padrão baseado em prefixos para facilitar a identificação:

`feat/`: Novas funcionalidades (ex: `feat/login-social`).
`fix/`: Correção de bugs (ex: `fix/erro-calculo-frete`).
`docs/`: Alterações em documentação (ex: `docs/readme-setup`).
`refactor/`: Refatoração de código existente.
`chore/`: Tarefas administrativas ou de build.
    
### 📝 Documentação de Repositórios
Todo repositório deve conter um `README.md` com:

Título e Descrição Clara.

Pré-requisitos e Instruções de Instalação.

Guia de Uso/Execução.

Explicação da Estrutura de Pastas (opcional, mas recomendado, pode ser apenas das principais).
